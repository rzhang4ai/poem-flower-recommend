## 一、核心设计原则

### 1.1 可解释性优先

本项目最大的技术贡献是**自训练的三层情感分类器**（SikuRoBERTa + FSPC + FCCPSL）和**双重确认意象体系**（CCPoem-BERT top10 ∩ 诗学含英），它们是去黑盒、可溯源的核心资产。

> 推荐顺序原则：**先用可解释的结构化特征做主要匹配，再用语义嵌入做补充桥接**，而非反过来。

### 1.2 分层匹配逻辑（四层）

```
第一层（必做）：情感极性向量匹配      ← 用你自训练的分类器
第二层（必做）：核心意象词 BM25 匹配   ← confirmed_imagery + sxhy_raw_words + ccpoem_top10
第三层（辅助）：赏析摘要嵌入匹配       ← shangxi_clean，现代话↔诗的语义桥
第四层（辅助）：场景 dim              ← 9维语义，提升场景相关性
```

### 1.3 多轮对话优先于单次表单

用户的一次自由输入往往不足以确定意图。系统应主动追问，而非静默降级。

## 二、用户意图分类与对话流程

### 2.1 顶层分类：送人 vs 自用

用户输入后，LLM 意图解析器（`llm_intent_parser.py`）**首先判断场合类别**：

| 场合类别 | 特征信号 | 核心追问方向 |
|---------|---------|------------|
| **送人** | "送给"、"赠"、"礼物"、"朋友"、"老师"、"情人"... | 对象是谁 + 关系 + 场合 |
| **自用** | "想看"、"自己"、"此刻心情"、"我想"... | 用户当前情感状态 + 核心意象 |
| **场合模糊** | 无明显人称 | 先追问「是送给别人还是自己欣赏？」 |

### 2.2 送人流程

```
用户：想找一首花诗送人
  ↓ 系统问：送给谁？（师长 / 朋友 / 伴侣 / 晚辈 / 其他）
用户：我的导师，他要退休了
  ↓ 系统提取：关系=师长，场合=退休，情感倾向=敬意+不舍
  ↓ 系统问：（若信息够）直接推荐；若花名偏好未知可问
```

**关系 → 推荐策略映射：**

| 关系 | 情感极性偏向（→ prob_l1 向量） | 场景 dim 偏向（→ dim_vector_idf 余弦） |
|------|-------------------------------|--------------------------------------|
| 师长 / 长辈 | 积极 / 隐性积极 | dim_virtue 权重拉高 |
| 朋友 / 同学 | 隐性消极（不舍）/ 积极（祝福） | dim_social 权重拉高 |
| 伴侣 / 爱人 | 隐性消极（相思）/ 积极 | dim_people 权重拉高 |
| 晚辈 | 积极 | dim_virtue 权重拉高 |
| 节日场合 | 积极 | dim_season 权重拉高 |

> **注**：这里只约定「用哪个维度的信号驱动」，不预设花名或意象词。最终推荐结果由情感向量余弦 + dim 余弦在全部 1075 首诗里实际排序得出，花名是数据说话，不是规则写死。

### 2.3 自用流程

```
用户：想找一首今晚的心情诗
  ↓ 系统问：现在的感觉更偏哪个方向？
  （展示 5 极性卡片，见 2.4）
用户：选择「不舒服」（隐性消极）
  ↓ 系统提取：情感极性 = implicit_negative → prob_l1 向量
  ↓ 系统问：你希望进入什么氛围？（可留空）
用户：想有雪或者月的感觉
  ↓ 系统提取：BM25 关键词 = 雪、月
  ↓ 推荐
```

### 2.4 情感引导卡片选项（5 极性，直接对应分类器）

直接使用情感分类器的 5 个极性标签，配 emoji 和日常口语标注，不做额外包装：

| 按钮 emoji | 按钮下方标注 | 分类器极性标签 | prob_l1 目标维度 |
|-----------|------------|--------------|----------------|
| 😄        | 很棒        | positive（积极） | `prob_l1_positive` 高 |
| 🙂        | 还 ok       | implicit_positive（隐性积极） | `prob_l1_implicit_positive` 高 |
| 😐        | 一般般       | neutral（中性） | `prob_l1_neutral` 高 |
| 😕        | 不舒服       | implicit_negative（隐性消极） | `prob_l1_implicit_negative` 高 |
| 😞        | 不好        | negative（消极） | `prob_l1_negative` 高 |

每个按钮点击后，直接用对应 prob_l1 维度置高的向量（如 `[0.0, 0.0, 0.0, 0.1, 0.9]` 对应"很棒"）做情感余弦匹配，不经过任何中间映射。

## 三、结构化数据与匹配通道

### 3.1 数据资产（`poems_structured_shangxi_wip.csv` 49 列）

| 来源 | 关键字段 | 推荐中的角色 |
|------|---------|------------|
| **情感分类器** | `prob_l1_*`（5 维）、`l3_c3_zh`（10 类） | **第一层**：与用户情感向量做余弦相似度 |
| **CCPoem-BERT** | `ccpoem_top10`、`confirmed_imagery` | **第二层 BM25 文档侧**：可解释意象词 |
| **诗学含英** | `sxhy_raw_words`、`sxhy_catalog_paths` | **第二层 BM25**；**第四层 dim**（路径→9维） |
| **赏析摘要** | `shangxi_clean`（本次新增） | **第三层嵌入**：现代话语义桥 |
| **9 维 dim** | `dim_vector_idf` | **第四层**：场景/关系语义向量 |

### 3.2 dim（9 维语义向量）怎么用

**dim 的本质**：把诗学含英的 40 个分类路径计数（哪一类出现多少次），再 IDF 加权归一化，得到该首诗的「语义侧重分布」。

**为何有用**：

- 用户说"送给老师"→ 系统推断场合偏向 `dim_virtue`（品格志向）→ 与诗的 `dim_vector_idf` 做余弦相似度，优先返回 `dim_virtue` 最强的诗（实际上就是咏梅/松/菊等象征品格的诗）。
- 用户说"送别朋友"→ 偏向 `dim_social`（人文社交，含祖饯类/送别类）。

**用户侧构建**：来自两种方式：
1. **场景预设**（`scene_presets.json`）：直接给定 9 维向量，精准、稳定。
2. **文本关键词**（`dim_keywords.json`）：用户话里扫描词典关键词。

**现阶段建议**：dim 作为**第四层辅助通道**（权重约 15-20%），主要在「用户明确说了场合/关系」时才激活，权重才较高；纯情绪描述时场景信号弱则置 0。

## 四、权重分配方案（修订版）

### 4.1 权重原则

- 情感和意象词是本项目**核心贡献**，优先级最高。
- 赏析嵌入是「现代话语义桥」，用于兜底，不应掩盖前两者。
- dim/atm 是辅助通道，按信号强度动态开关。

### 4.2 分场景权重

| 通道 | 无场景线索 | 有场景/关系线索 | 纯情绪描述 |
|------|-----------|--------------|-----------|
| **BM25 意象词** | 0.35 | 0.25 | 0.40 |
| **情感向量余弦** | 0.35 | 0.25 | 0.45 |
| **赏析嵌入** | 0.20 | 0.20 | 0.15 |
| **9 维 dim** | 0.0 | 0.20 | 0.0 |
| **6 维气氛** | 0.10 | 0.10 | 0.0 |

所有通道**无信号时自动置 0**，其余按比例归一化至总权重 1.0。

### 4.3 RRF 参数建议

每个通道输出 **排序列表**（而非分数），通过 Reciprocal Rank Fusion 合并。`k=60` 为常规值；候选集过小（< 30）时可降至 30。

## 五、多轮对话状态机

```
状态 0：等待用户输入
  ↓ 收到输入
状态 1：LLM 意图解析（llm_intent_parser.parse）
  ↓ 判断是否信息完整
  ├─ 不完整 → 状态 2：追问
  └─ 完整   → 状态 3：推荐
状态 2：追问（最多 2 轮）
  ↓ 收到补充
  → 更新 IntentResult，回到状态 1 判断
状态 3：推荐结果展示（默认 3 条）
  ↓ 用户反馈 / 精调
  ├─ 精调场景/情感 → 重新加权，更新推荐
  └─ 满意 → 结束
```

**「信息完整」判断条件：**

| 场合 | 完整条件 |
|------|---------|
| 送人 | 关系已知（师长/朋友/伴侣/…）+ 情感基调已知 |
| 自用 | 情感极性已知（5 选 1）|
| 两者 | 意象线索有无不限（无则范围宽泛但仍可推荐） |

## 九、关于 dim 维度的补充解释

> 供自用和对外说明参考

**dim 是什么**：将诗学含英 40 个小类按计数映射为 9 个高层维度，IDF 加权后 L2 归一化。它不是情感，不是字面意象，而是「这首诗的语义重心落在哪里」的结构化描述。

**9 个维度含义：**

| 维度 | 含义 | 典型诗 |
|------|------|--------|
| dim_nature | 自然天象（月/风/雪） | 写景咏物诗 |
| dim_season | 时令节序（春/重阳） | 节令诗 |
| dim_space | 地理空间（山/楼/江） | 山水诗、边塞诗 |
| dim_people | 人物关系（闺中/君臣） | 闺怨诗、宫廷诗 |
| **dim_virtue** | **品格志向（气节/隐逸）** | **咏梅/菊/竹高洁诗**（IDF 最高=最有区分力） |
| dim_artifact | 器物生活（琴/舟/酒） | 隐逸生活诗 |
| dim_biota | 动植物（鸟/草/树） | 咏物诗 |
| **dim_social** | **人文社交（送别/庆吊）** | **送别诗、赠人诗**（IDF 第二高） |
| dim_flower | 花卉意象（百花类） | 本数据集几乎都有，单独拆出以降低噪音 |

**关键洞察**：`dim_virtue`（IDF=2.34，仅 10% 诗命中）和 `dim_social`（IDF=1.45，24% 命中）区分力最强——这两个正好对应「赠师长」和「送别」两大最高频送花场景，这是 dim 在本系统里最大的价值所在。
