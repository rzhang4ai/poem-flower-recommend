## 一、BERT 模型选型

### 1.1 古诗词领域预训练模型全景

| 模型 | 训练语料 | 核心优势 | 最适合任务 | 获取 |
|------|---------|---------|-----------|------|
| **BERT-CCPoem** (清华 THUNLP) | 926,024首古诗 CCPC-Full v1.0 | 诗句语义相似度最强 | 语义相似度、embedding提取 | GitHub THUNLP-AIPoet |
| **SikuRoBERTa** (南农大) | 《四库全书》全文 ~400M字 | 古文覆盖最广，RoBERTa架构 | 古文分类、情感分析 | HuggingFace SIKU-BERT/sikuroberta |
| **AnchiBERT** (电子科大) | 3950万字文言文 | 诗歌主题分类 SOTA | 主题分类、风格分析 | GitHub ttzHome/AnchiBERT |
| **bert-ancient-chinese** | ~6倍四库全书 | 词表最大(38208词) | 生僻字处理 | HuggingFace Jihuai/bert-ancient-chinese |
| **BERT-base-Chinese** (Google) | 现代中文 Wikipedia | 对赏析文本效果好，最稳定 | 赏析文本分析 | HuggingFace google-bert |

> **选型决策：双模型策略**
> - 正文（古文）→ BERT-CCPoem 或 SikuRoBERTa
> - 赏析（现代中文）→ BERT-base-Chinese 或 SikuRoBERTa
> - 两路 embedding 分别计算后加权合并

### 1.2 BERT-CCPoem 能做什么、不能做什么

阅读 README 和相关论文，须明确边界：

| 功能 | 支持情况 | 说明 |
|------|---------|------|
| 生成句子级 embedding | ✅ 直接支持 | `gen_vec_rep.py` 输出 768维向量 |
| 计算诗句语义相似度 | ✅ 直接支持 | 余弦相似度，显著优于 TF-IDF |
| 情感分类（8维） | ❌ 需 fine-tune | 无分类头，需接 FSPC 数据集微调 |
| 意象词提取 | ⚠️ 间接支持 | 用 attention 权重定位重要 token |
| 主题分类 | ❌ 需 fine-tune | AnchiBERT 在此任务有 SOTA，更推荐 |
| 花卉象征关系 | ❌ 需构建 | embedding 聚类可发现语义邻近，但需人工验证 |

### 1.3 BERT 如何替代/增强传统方法

| 传统方法 | 局限性 | BERT 增强方案 |
|---------|--------|--------------|
| TF-IDF 相似度 | 词袋模型，「高洁」和「纯洁」视为不同词 | BERT-CCPoem sentence embedding 做余弦相似度，捕获语义等价 |
| TextRank 关键词 | 基于共现窗口，无上下文理解 | attention 权重替代 PageRank 投票 |
| PMI 花名-词共现 | 200条样本偏少，统计不稳定 | BERT contextual embedding 计算词-花名语义相似度，弥补小样本不足 |
| k-Means（TF-IDF空间） | 高维稀疏，语义损失 | 用 BERT embedding（768维稠密）作聚类输入，语义凝聚力更强 |

**`bert_embed.py` 核心逻辑：**

```python
from transformers import AutoTokenizer, AutoModel
import torch, numpy as np, pandas as pd, json

# 正文：BERT-CCPoem（按 GitHub 说明下载到本地）
tok_poem  = AutoTokenizer.from_pretrained('./bert_ccpoem')
mdl_poem  = AutoModel.from_pretrained('./bert_ccpoem')

# 赏析：SikuRoBERTa（HuggingFace 直接下载）
tok_ana   = AutoTokenizer.from_pretrained('SIKU-BERT/sikuroberta')
mdl_ana   = AutoModel.from_pretrained('SIKU-BERT/sikuroberta')

def get_embedding(text, tok, mdl):
    """取 [CLS] token 最后一层 hidden state 作为句子 embedding"""
    if not text or not str(text).strip():
        return np.zeros(768)
    inputs = tok(str(text), return_tensors='pt',
                 max_length=512, truncation=True, padding=True)
    with torch.no_grad():
        out = mdl(**inputs, output_attentions=True)
    emb = out.last_hidden_state[:, 0, :].squeeze().numpy()
    return emb

def get_token_importance(text, tok, mdl, top_k=10):
    """
    用最后一层 attention 权重均值作为 token 重要性分数
    返回 top_k 个重要 token（排除 [CLS][SEP] 和标点）
    """
    inputs = tok(str(text), return_tensors='pt',
                 max_length=512, truncation=True)
    tokens = tok.convert_ids_to_tokens(inputs['input_ids'][0])
    with torch.no_grad():
        out = mdl(**inputs, output_attentions=True)
    # shape: (num_heads, seq_len, seq_len) → 取列均值作为每个token的重要性
    attn = out.attentions[-1].squeeze().mean(dim=0).mean(dim=0).numpy()
    scored = [(tok_str, float(score))
              for tok_str, score in zip(tokens, attn)
              if tok_str not in ['[CLS]','[SEP]','[PAD]','，','。','！','？','、']]
    scored.sort(key=lambda x: x[1], reverse=True)
    return scored[:top_k]

# 主循环
df = pd.read_csv('../step0_sample/output/sample_200.csv')
poem_embs, ana_embs, token_imp_rows = [], [], []

for _, row in df.iterrows():
    poem_embs.append(get_embedding(row['正文'], tok_poem, mdl_poem))
    ana_embs.append(get_embedding(row['赏析'], tok_ana, mdl_ana))
    imp = get_token_importance(row['正文'], tok_poem, mdl_poem)
    token_imp_rows.append({
        'ID': row['ID'], '花名': row['花名'], '月份': row['月份'],
        'top10_tokens': json.dumps(imp, ensure_ascii=False),
        'top5_preview': '、'.join([t for t,_ in imp[:5]])
    })

np.save('output/bert_ccpoem_embeddings.npy', np.array(poem_embs))
np.save('output/bert_analysis_embeddings.npy', np.array(ana_embs))
pd.DataFrame(token_imp_rows).to_csv('output/bert_token_importance.csv',
                                     index=False, encoding='utf-8-sig')
print(f"✅ embeddings: {np.array(poem_embs).shape}")
```

> **计算资源**：BERT-CCPoem 约 390MB，CPU 推理 200 条约 5-10 分钟。Mac Mini M4 有 Metal GPU 加速，建议在 M4 上运行。可加 `device = torch.device('mps')` 并将模型移到 mps。

## 二、情感分析替代方案

### 2.1 SnowNLP 的根本问题

| 问题 | 说明 |
|------|------|
| 训练语料偏差 | 训练自淘宝/大众点评，语言风格与古典诗词完全不同 |
| 词汇表不匹配 | 「凌寒」「傲霜」「高洁」等在商品评论中几乎不存在 |
| 含蓄表达失效 | 古诗以意象寄情（「折柳」=离别），SnowNLP 看不懂隐性情感 |
| 粒度太粗 | 只有正/负/中性，古诗词情感需要 5-8 维细分 |

## 三、意象提取

### 3.1 意象的精确定义（区别于关键词）

「意象」不是简单的高频词或关键词，它需要同时满足：

1. **是具体物象**（梅/月/东篱/折柳）而不是抽象词（美好/哀愁）
2. **携带文化情感记忆**——在历史诗歌使用中已积累了情感内涵
3. **在当前语境中被激活**——诗人有意用它承载情感

> 「桌子」是普通名词，「东篱」（陶渊明归隐田园的具体场所）才是意象——它承载了整个「归隐田园」的文化情感记忆。

---

### 3.2 任务 A：每首诗 top-5 意象

**两层判断**：先识别「是物象」，再判断「有意象价值」

#### 方法一：规则词典 + TF-IDF×PMI 排序（立即可实现）

**三类意象词典**：

**提取逻辑（新建 `step2c_imagery.py`）**：

```python
def extract_poem_imagery(row, tfidf_scores, pmi_scores, top_k=5):
    """
    对每首诗提取 top-k 意象
    得分 = TF-IDF权重 × PMI分数 × 意象词典权重(1.5)
    """
    try:
        tokens = json.loads(row['tokens_poem'])
    except:
        tokens = []

    scored = []
    for token in set(tokens):  # 去重
        if token not in IMAGERY_VOCAB:
            continue
        tfidf_w = tfidf_scores.get(token, 0.01)
        pmi_w   = pmi_scores.get((row['花名'], token), 0.0)
        # PMI 可以为负（词与该花负相关），取 max(0, pmi)
        score = tfidf_w * (1 + max(0, pmi_w)) * 1.5
        scored.append((token, round(score, 4)))

    scored.sort(key=lambda x: x[1], reverse=True)
    return scored[:top_k]
```

#### 方法二：BERT-CCPoem attention 权重（语义增强）

```python
def extract_imagery_by_attention(row, tok, mdl, imagery_vocab, top_k=5):
    """
    用 BERT attention 权重排序意象词
    先用词典过滤候选，再用 attention 决定最终排序
    """
    text = str(row.get('正文', ''))
    inputs = tok(text, return_tensors='pt', max_length=128, truncation=True)
    tokens = tok.convert_ids_to_tokens(inputs['input_ids'][0])

    with torch.no_grad():
        out = mdl(**inputs, output_attentions=True)
    # 最后一层所有头的平均 attention
    attn = out.attentions[-1].squeeze().mean(0).mean(0).numpy()

    candidates = []
    for tok_str, score in zip(tokens, attn):
        if tok_str in imagery_vocab:
            candidates.append((tok_str, float(score)))

    candidates.sort(key=lambda x: x[1], reverse=True)
    # 去重（BERT 可能将同一个词拆成多个 token）
    seen, result = set(), []
    for tok_str, score in candidates:
        if tok_str not in seen:
            seen.add(tok_str)
            result.append((tok_str, score))
        if len(result) >= top_k:
            break
    return result
```

> **最终策略**：方法一（词典×TF-IDF×PMI）得到候选集，方法二（BERT attention）决定排序。两者的交集作为高置信度意象。

**输出格式**：

| 字段 | 说明 |
|------|------|
| `imagery_poem_top5` | 正文提取的核心意象，如 `['梅花','凌寒','傲雪','暗香','疏影']` |
| `imagery_analysis_top5` | 赏析中提取的意象（通常与正文互补）|
| `imagery_bert_top5` | BERT attention 版本 |
| `imagery_method` | 使用的方法标记 |

## 四、意象 vs 象征：概念辨析与知识图谱设计

### 4.1 本质关系

> **一句话：象征 = 意象的历史统计平均 + 文化认可的固化结果**

| 维度 | 意象 (Imagery) | 象征 (Symbolism) |
|------|---------------|-----------------|
| 存在形式 | 动态的，语境依赖 | 静态的，文化固化 |
| 来源 | 从具体诗文中提取 | 从辞书、大量文本统计中抽取 |
| 颗粒度 | **诗句级**（每首诗有自己的意象） | **花种级**（梅花作为整体的象征）|
| 在本项目中 | 每首诗的 `imagery_poem_top5` | 每种花的 `symbolism` 标注维度 |
| 随时间变化 | 会因诗人个性而变 | 相对稳定，但朝代间有演变 |

**典型案例**：
- 「梅」在陆游诗中是「孤高/不随流俗」意象（个性语境）
- 「梅」在王安石诗中是「报春先知/不畏严寒」意象（个性语境）
- **「梅花 → 傲寒不屈/高洁」= 象征**（两者共有的核心 + 历史积累公认）

---

### 4.2 按朝代分布的意象/象征知识图谱

#### 为什么朝代维度重要

Gong & Zhou (2025, arXiv:2505.04785) 已经用 BERT 检测了**唐宋两代**牡丹和梅花的情感演变（仅 2 种花 × 2 个朝代）。我们可以扩展为 **79 种花 × 15 个朝代**的完整矩阵——这在学术上是显著的原创贡献。

**实际上这个图谱能揭示的规律**：
- 「梅花」在**宋代**偏向「清瘦/暗香/孤高」，在**唐代**偏向「凌寒/傲骨/报春」
- 「荷花」在**魏晋**强调「出淤泥而不染/君子」，在**宋代**多了「愁/离别」的意味
- 「菊花」从**东晋**（陶渊明）开始就与「东篱/归隐」强绑定，此后历代延续

#### 三层图谱结构

```
第一层：花-朝代层
  节点：花名（79个）+ 朝代（15个）
  边：「主要出现于」，权重=该花在该朝代的诗词数量
  → 能看到：哪些花是哪个朝代的「主打花」

第二层：花-意象层（按朝代着色）
  节点：花名 + 意象词
  边：「在X朝的意象关联」
    权重 = 该朝代内的 PMI 分数
    颜色 = 朝代（唐=金色，宋=蓝色，元=绿色…）
  → 能看到：同一种花的意象群在不同朝代如何演变

第三层：意象-情感层
  节点：意象词 + 情感标签（8类）
  边：「关联情感」，权重=情感词典分数
  → 能看到：凌寒→bold+elegant；清瘦→elegant+sorrow
```

```python
# step_kg/build_knowledge_graph.py
import pandas as pd, json
from collections import defaultdict

def build_flower_dynasty_imagery_graph(df, pmi_by_dynasty, imagery_vocab):
    """
    构建 (花名, 朝代, 意象词, PMI分数) 四元组
    是整个知识图谱的数据基础
    """
    DYNASTY_ORDER = ['先秦','汉','魏晋','南北朝','隋','唐','五代','宋','元','明','清']
    triples = []

    for dynasty in DYNASTY_ORDER:
        subset = df[df['朝代'].str.contains(dynasty, na=False)]
        if len(subset) < 3:
            continue
        # 在该朝代子集内重新计算 PMI
        pmi = compute_pmi_subset(subset, imagery_vocab)
        for flower, word_scores in pmi.items():
            for word, pmi_score in word_scores[:10]:
                triples.append({
                    'flower': flower,
                    'dynasty': dynasty,
                    'imagery': word,
                    'pmi': pmi_score,
                    'doc_count': len(subset[subset['花名']==flower])
                })

    return pd.DataFrame(triples)

def export_to_pyvis(triples_df, output_path='knowledge_graph.html'):
    """导出为可交互 HTML（pyvis）"""
    from pyvis.network import Network
    net = Network(height='800px', width='100%', bgcolor='#1a1a2e', font_color='white')

    DYNASTY_COLORS = {
        '唐': '#FFD700', '宋': '#4169E1', '元': '#228B22',
        '明': '#DC143C', '清': '#FF8C00', '魏晋': '#9370DB',
    }

    # 添加花名节点（大节点）
    for flower in triples_df['flower'].unique():
        count = triples_df[triples_df['flower']==flower]['doc_count'].max()
        net.add_node(flower, label=flower, size=min(30, 10+count//3),
                     color='#E8C547', title=f'花卉：{flower}')

    # 添加意象词节点（小节点，颜色=朝代）
    for _, row in triples_df.iterrows():
        node_id = f"{row['imagery']}_{row['dynasty']}"
        color = DYNASTY_COLORS.get(row['dynasty'], '#AAAAAA')
        net.add_node(node_id, label=row['imagery'], size=8,
                     color=color, title=f"{row['dynasty']}·{row['imagery']} (PMI={row['pmi']:.2f})")
        net.add_edge(row['flower'], node_id, value=row['pmi'],
                     title=f"PMI={row['pmi']:.2f}，{row['dynasty']}")

    net.save_graph(output_path)
```

### 5.4 Symbolism Rules 具体修改

现有象征词典是硬编码的。升级方向：以 `pmi_flower_word.csv` 的输出数据为依据动态更新：

| 花名 | 现有（硬编码）| 建议根据 PMI 数据补充 |
|------|-------------|---------------------|
| **梅花** | 坚韧/高洁/傲寒/不屈 | 孤高/暗香/疏影/清冷/玉肌/铁骨（宋词意象群）|
| **荷花** | 高洁/纯洁/清廉 | 凉风/晓露/碧叶/君子/遗爱（「遗爱」出自《爱莲说》）|
| **菊花** | 隐逸/清雅/傲霜/高洁 | 陶潜/东篱/南山/幽香/黄花/晚节（晚节=品格坚守到晚年）|
| **牡丹** | 富贵/繁荣/国色/雍容 | 天香/倾城/国艳/春风/帝城/骄艳（唐诗固定意象群）|
| **桃花** | 爱情/美人/春天/桃源 | 烂漫/夭夭/灼灼/仙境/避世/不知秦汉（两个典故系统）|

```python
# 从 pmi_flower_word.csv 动态更新象征词典的方法

def update_symbolism_from_pmi(pmi_df, imagery_vocab, pmi_threshold=1.5):
    """
    用 PMI 数据增强象征词典
    只添加同时满足：PMI > threshold 且在意象词典中的词
    """
    updates = {}
    for _, row in pmi_df.iterrows():
        flower = row['花名']
        top_words = json.loads(row['pmi_top_words'])
        # 筛选：高PMI 且是意象词
        symbolism_candidates = [
            w for w, score, cnt in top_words
            if score > pmi_threshold and w in imagery_vocab and cnt >= 2
        ]
        updates[flower] = symbolism_candidates[:8]  # 最多补充8个
    return updates
```
